1. Purpose & Audience

This dashboard is a self-contained, single-file HTML report used by harpin.ai leadership during the monthly revenue review meeting. It provides a complete, real-time view of the agreement portfolio — milestone billing status, invoice timeliness, amount variances, and contract health.

Primary Audience

  • Head of Operations
  • Finance / Revenue team
  • Executive leadership

Review Cadence

Generated once per month, covering the prior month close (e.g., March) and the current month in-progress (e.g., April). The report date drives all "current month" and overdue logic.

3. Agreement Inclusion Rules

Not all agreements in Fibery are displayed. Apply these filters:

RuleAction
Agreement type = InternalExclude (e.g., "harpin General")
"test" anywhere in the agreement name (case-insensitive)Exclude
All other agreementsInclude regardless of workflow state
Note: Contract Complete agreements ARE included — they appear in the portfolio table and milestone detail, but are excluded from expiry alerts (see Section 5).

4. Dashboard Sections

The dashboard renders as a single scrollable HTML page with the following sections in order:

4.1 Page Header

Dark navy banner showing: harpin.ai — Monthly Revenue Review, the review month/year, generation date, agreement count, and data source.

4.2 Executive Summary KPIs

Six metric cards in a horizontal grid:

CardMetricColor Rule
Total Portfolio ValueSum of Target Revenue across all included agreementsNavy (neutral)
Revenue RecognizedSum of Rev Recognized + % of portfolioGreen (always)
Prior Month CloseAmount invoiced in prior month + milestone countBlue (neutral)
Current Month InvoicedAmount invoiced so far this month + % of milestonesGreen if ≥50%, Yellow if <50%
Amount VariancesCount of milestones with non-zero variance (March/April only, non-Not-Scheduled only)Yellow if >0, Green if 0
Overdue / At-RiskCount of milestones past target date, not invoiced, >$0Red if >0, Green if 0

4.3 Agreement Expiry Alerts

Only shown if there are expired or ending-soon agreements. Two alert types:

  • Expired (red): End date is in the past AND workflow state is NOT Contract Complete
  • Ending soon (yellow): End date is within 90 days AND state is NOT Contract Complete
Important: Contract Complete agreements are never shown in expiry alerts, even if their end date has passed.

4.4 Prior Month Billing Review

Full table of all milestones with a Target Date in the prior review month (e.g., March 2026). Not Scheduled milestones are excluded. Columns:

  • Agreement name · Milestone · Target Date · Target $ · Actual Date · Actual $ · On Time? · Variance · Status badge

4.5 Current Month Milestones

Same structure as Section 4.4 but filtered to the current month (e.g., April 2026). Not Scheduled milestones are excluded. Header shows count invoiced so far and total target for the month.

4.6 Agreement Portfolio

Full table of all included agreements, sorted by customer revenue (descending). Columns:

  • Customer · Agreement name · Type · Status · Start · End Date · Target Revenue · Recognized · % Collected (progress bar) · Milestones (invoiced/total)

Agreements with variances (per Section 5 rules) show an orange "X variance" mini-badge next to the name.

4.7 Revenue by Customer

Rollup table sorted by total contract value (descending). Columns: Customer · Agreement count · Total Contract Value · Revenue Recognized · Remaining · % Collected. Includes a bold TOTAL row.

4.8 Overdue / At-Risk Milestones

Only shown if there are overdue items. Lists all milestones where: Target Date < today AND state ≠ Invoiced AND Revenue Recognized = False AND Target Amount > $0.

4.9 Full Milestone Detail

Collapsible <details> elements, grouped by customer then agreement. Each agreement shows all its milestones with the same columns as the monthly tables. Click header to expand/collapse. Summary line shows invoiced count, target revenue, and recognized revenue.

4.10 Sortable Columns

All table headers are clickable to sort ascending (▲) or descending (▼). Sorting handles money, dates, percentages, and text. Total rows stay pinned to the bottom.

5. Business Logic & Calculation Rules

5.1 Variance Eligibility

A milestone counts as a variance item (for KPI card, portfolio badges, and table indicators) only if ALL of the following are true:

  • Amount Variance ≠ 0
  • Target Date falls in the current or prior review month (e.g., 2026-03 or 2026-04)
  • Workflow state ≠ Not Scheduled
Variances from outside the two-month review window (e.g., a Month 1 prorated billing from November 2024) are intentionally excluded. This keeps the report focused on the current review period.

5.2 On-Time Indicator

For invoiced milestones only (state = Invoiced OR Revenue Recognized = True):

  • ✓ On Time — Actual Date ≤ Target Date
  • ⚠ Late — Actual Date > Target Date (shows days late)
  • — (dash) — Not yet invoiced, or missing date data

5.3 Milestone Status Badge

ConditionBadge
Invoiced or Recognized, on time, no variance✓ On Time
Invoiced or Recognized, but late and/or has variance⚠ Late / Variance $X
Scheduled, target date in the futureScheduled
Scheduled, target date has passed⚠ Overdue
Not Scheduled, target date has passed✗ Not Scheduled (Overdue)
Not Scheduled, target date in the futureNot Scheduled
Not Delivered, Target Amount = $0N/A ($0)
Not Delivered, Target Amount > $0✗ Not Delivered

5.4 Expiry Logic

  • Agreements with state = Contract Complete are never flagged as expired or ending soon — regardless of end date.
  • Expired = end date < today AND state ≠ Contract Complete
  • Ending soon = end date is within 90 days (but not past) AND state ≠ Contract Complete

5.5 Overdue Definition

A milestone is overdue if ALL of:

  • Target Date < today (report generation date)
  • Workflow state ≠ Invoiced
  • Revenue Recognized ≠ True
  • Target Amount > $0 (excludes $0 placeholder milestones)

5.6 Portfolio % Collected

Driven by the agreement-level Rev Recognized field, not by counting invoiced milestones. This can differ from the milestone invoiced count when revenue is pre-recognized or when actual amounts differ from targets.

7. Data Freshness & Refresh

The generator script caches revenue items in /tmp/revenue_items.json and agreements in /tmp/agreements_clean.json. These are session-scoped and need to be re-fetched when rebuilding from scratch.

Selective Refresh (within a session)

Individual agreements can be refreshed without a full rebuild by fetching revenue items for specific agreement IDs and replacing their entries in the cache. This was used during the April 2026 review to update Order Form 2 (LeadWhisper Active Users) and T&L agreements mid-session.

When to do a full re-fetch: At the start of each monthly review session, always re-fetch both the agreements list and all revenue items from Fibery to ensure the data is current.

8. Known Data Considerations

8.1 Revenue Recognized vs. Invoiced State

These are tracked separately in Fibery. A milestone can have Revenue Recognized = True while its workflow state is still "Scheduled" (not yet invoiced). This is most common in flat-rate subscriptions where amounts are pre-entered and recognized is checked in advance.

Watch for: Future-dated milestones (months that haven't occurred yet) with Revenue Recognized = True and actual amounts pre-filled. This inflates the agreement-level Rev Recognized figure and the portfolio % complete. Example found in April 2026 review: Bartaco Order Form 2 Answers Subscription had months 13–21 (May 2026–Jan 2027) pre-marked as recognized, overstating recognized revenue by ~$16,000.

8.2 $0 Query Usage Milestones

Order Form 2 - LeadWhisper Active Users (PCL) carries parallel Active Users and Query Usage milestones each month. In the early months (1–4), Query Usage was priced at $0 (not yet activated) with state "Not Delivered." This is expected and not an error — the $0 / Not Delivered combination renders as N/A ($0).

8.3 Prorated First Month Variances

Agreements that start mid-month often have a prorated target amount for Month 1. If billed at the full monthly rate, this creates a positive variance. Example: Bartaco Order Form 1 Tools Access Month 1 (Nov 14–30, 2024) — target $1,960 (prorated), actual $3,500 (full month), variance +$1,540. No note or error flag was present in Fibery. This variance is outside the March/April window so it does not appear in the variance report.

8.4 Agreement End Dates vs. Contract State

An agreement's end date in the Duration field may be in the past even when it's in good standing. Always check workflow state before treating an agreement as "expired." Agreements in Contract Complete state should never be shown in expiry alerts.

8.5 Invoice Count vs. Revenue % Mismatch

The milestone invoiced count (e.g., 12/24) and the revenue % complete (e.g., 83%) can diverge significantly. This is by design — the % is driven by the agreement-level Rev Recognized field which can be pre-populated, while the count reflects the actual number of milestones moved to Invoiced state.

10. Rebuild Instructions

To rebuild the dashboard from scratch in a new session, give Claude the following prompt (update the month/year as needed):

Build the harpin.ai monthly revenue review dashboard for [Month Year].

Use the PRD at harpin Main/Revenue Review/PRD/revenue-review-dashboard-PRD.html for full spec.

Steps:
1. Fetch Fibery credentials from macOS Keychain:
   - API token: security find-generic-password -s fibery -a api-token -w
   - Workspace URL: security find-generic-password -s fibery -a workspace-url -w

2. Fetch all agreements from Agreement Management/Agreements (exclude Internal type
   and any with "test" in the name).

3. Fetch all revenue items from Agreement Management/Revenue Item (all 500, use
   pagination if needed).

4. Generate the HTML dashboard per the PRD spec and save to:
   harpin Main/Revenue Review/Monthly/revenue-review-[month]-[year].html

Review month = [prior month, e.g. March 2026]
Current month = [current month, e.g. April 2026]
Report date = [today's date, e.g. April 14, 2026]

Mid-Session Agreement Refresh

If an agreement has been updated in Fibery during the review meeting and needs to be refreshed without a full rebuild:

Re-fetch the revenue items for [Agreement Name] from Fibery and regenerate the HTML.
The agreement ID is [fibery/id].
