1. Purpose & Audience

This dashboard is a self-contained, single-file HTML report that shows every employee who logged under or over 40 hours in the previous work week, using time entry data from Fibery. It is generated on demand by running a single Python script.

Primary Audience

  • Head of Operations
  • People / HR team
  • Team leads reviewing utilization

What it answers

  • Who logged more than 40 hours last week, and by how much?
  • Who logged fewer than 40 hours last week, and by how much?
  • Which projects did each person log time to?

2. Data Source & Credentials

Fibery workspace at https://harpin-ai.fibery.io. All data comes from the Agreement Management/Labor Costs entity type, which is populated via Clockify integration.

Date Filter (confirmed working syntax)

{
  "command": "fibery.entity/query",
  "args": {
    "query": {
      "q/from": "Agreement Management/Labor Costs",
      "q/select": [ ... ],
      "q/where": ["q/and",
        [">", "Agreement Management/Start Date Time", "$start"],
        ["<", "Agreement Management/Start Date Time", "$end"]
      ],
      "q/limit": 1000,
      "q/offset": 0
    },
    "params": {
      "$start": "2026-04-05T23:59:59Z",
      "$end":   "2026-04-13T00:00:00Z"
    }
  }
}
Note: Inline date literals in q/where do NOT work. You must use named params. The q/order-by clause also does not support datetime fields — sort in Python after fetching.

Pagination

Fibery returns a maximum of 1000 records per request. The script loops using q/offset (0, 1000, 2000…) until an empty result is returned. A typical week returns ~500 entries across ~55 employees.

4. Date Range Logic

The script dynamically computes "last week" relative to the date it is run. The work week is defined as Monday through Sunday.

import datetime
from datetime import timedelta

today      = datetime.date.today()
# weekday(): Mon=0, Tue=1, ... Sun=6
week_start = today - timedelta(days=today.weekday() + 7)  # last Monday
week_end   = week_start + timedelta(days=7)               # this Monday (exclusive)

# Fibery params — start is end-of-day Sunday before week_start
param_start = (week_start - timedelta(days=1)).strftime('%Y-%m-%dT23:59:59Z')
param_end   = week_end.strftime('%Y-%m-%dT00:00:00Z')

Example: run on Wednesday April 15, 2026 → week_start = 2026-04-06 (Mon), week_end = 2026-04-13 (Mon exclusive), meaning the dashboard covers Mon Apr 6 – Sun Apr 12.

5. Aggregation & Classification Rules

Grouping

Records are grouped by Agreement Management/Time Entry User Name (case-sensitive, as returned by Clockify). Each group accumulates:

  • Total hours — sum of Agreement Management/Hours (cast to float)
  • Entry count — number of individual time log records
  • Project set — unique values of Agreement Management/Time Entry Project Name

Classification

CategoryRuleColorSort Order
Over 40htotal hours > 40.0RedHighest hours first
Under 40htotal hours < 40.0Yellow / AmberLowest hours first
Exactly 40htotal hours == 40.0GreenAlphabetical
Floating-point note: Hours are summed from decimal strings (e.g. "8.0", "1.5"). The exact-40 bucket uses == 40.0 comparison. If floating-point drift is ever observed, switch to rounding to 2 decimal places before comparison.

9. Known Data Considerations

Employee name format

Names come directly from Clockify and are inconsistent — some are full names (Jordan Meyer), others are email prefixes (kelsey.hoff), and some are first-name only (Max, josh). The dashboard displays them as-is. No normalization is applied.

Exclusion list

The EXCLUDE_NAMES set near the top of the script suppresses specific people from all sections of the dashboard. This is used for two reasons:

  • Duplicate Clockify accounts — some employees have two records (e.g. a dot-format and a space-format name). The canonical record is kept; the duplicate is excluded.
  • Offboarded employees — people who have left harpin but whose Clockify/Fibery records have not yet been deactivated.

Matching is case-insensitive against the exact lowercase name string. To add or remove someone, edit the set in the script and rerun.

Data is not ordered in Fibery

q/order-by does not work on datetime fields in this workspace. The script fetches all matching records unordered and sorts in Python after aggregation.

Entries span multiple pages

A typical week has ~500 entries spread unevenly across the 8000+ total records in the type. The script always paginates until the API returns an empty page — do not assume all records fit in the first 1000.

Timezone

Start Date Time is stored in UTC. The filter boundary is set to end-of-day (23:59:59Z) on the Sunday before the week starts and midnight (00:00:00Z) on the Monday after the week ends. This correctly captures entries for US-based employees whose local midnight may differ by up to 8 hours.

Future entries

Clockify occasionally sync entries dated slightly in the future due to timezone offsets. These will be captured if their UTC Start Date Time falls within the filter window — this is expected and harmless.

10. Rebuild Instructions

To regenerate the dashboard for the most recent completed week, paste the following prompt into Claude Code from the harpin Main working directory:

Rebuild prompt (copy exactly):

Run python3 "Inbox/generate_labor_hours.py" and open the output HTML file.

That's it — the script handles everything: credential retrieval, Fibery query, pagination, aggregation, classification, HTML generation, and saving the file.

To rebuild with changes to the script

  1. Edit Inbox/generate_labor_hours.py
  2. Run: python3 "Inbox/generate_labor_hours.py"
  3. Open the output HTML to verify

To change the week (run for a specific past week)

Replace the dynamic date block near the top of the script with hardcoded values:

# Replace the dynamic date block with:
week_start = datetime.date(2026, 3, 30)  # the Monday of the desired week
week_end   = week_start + timedelta(days=7)

Then re-run. Remove the override afterward to restore dynamic behavior.

To add a new column (e.g., billable hours only)

  1. Add the field to the q/select list in the query
  2. Track the new value in the employees aggregation dict
  3. Add a column to the HTML table headers and the row-building function

Manual verification

After regenerating, spot-check two or three employees against their Clockify timesheet or Fibery records at https://harpin-ai.fibery.io → Agreement Management → Labor Costs.
